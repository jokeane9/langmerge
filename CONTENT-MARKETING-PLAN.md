# Langmerge — Content Marketing Plan

*Working doc — last updated 2026-06-27. The plan for how Langmerge + the personal builder brand show up, build credibility, and feed the pipeline.*

---

## 1. Positioning (who we are)

- **Langmerge = the WHAT** (agentic product studio + services). **John = the WHO** — lead the brand/media with the WHO.
- **The lane** (a rare three-way intersection): **practitioner, not pundit** (ships real systems, doesn't just talk) × **AI/agent velocity** (a seasoned engineer who builds fast with agentic tooling — out-delivers per week) × **human-in-the-loop realism** (knows where agents break and where a human stays). E-commerce/SMB is the *proof vertical*, not the headline.
- **The authentic WHO:** a seasoned, AI-fluent engineer who **drops into existing tech teams as a senior/embedded dev and ships features** — builds fast with agentic tooling, owns a real scope, and is honest about where the human has to stay in the loop. Still-hungry operator with a pulse on new dev practices (builds with AI, not hype). The products (Shelf, Vizidex, the WC agent) are *proof he ships*, not the headline. Ambition: **win, not be a billionaire** (craft, good clients, a reputation that compounds).
- **The one-liner:** *"I drop into tech teams as a senior engineer and ship features fast with AI/agent tooling — and I'm honest about where the agents break and where a human has to stay in the loop."*

## 2. The two-channel split

- **LinkedIn → the professional / build-in-public side.** Founder- and client-facing. Story + takeaway. This is the Langmerge pipeline + personal builder brand. *(LinkedIn is for the studio/builder voice — NOT the plugins.)*
- **killdate.dev → the build / credibility side.** Deeper, technical engineering write-ups for devs, peers, and eng leaders. The guts.
- **Same raw material, split by depth** — write once, pitch at the altitude each audience cares about.

## 3. Two kinds of content (don't confuse them)

- **Build documentation (credibility) — THE #1 LEVER → killdate.dev.** Detailing how we actually architected and shipped things: decisions, tradeoffs, what broke, how we reasoned. *Sells YOU to teams/peers* and **directly earns embedded/drop-in work** (eng leaders trust demonstrated craft). Unfalsifiable, and a cheap byproduct of work already being done.
- **Product marketing (sales).** Plugin pages, CAC, conversion copy. *Sells products to customers.* Keep it minimal and separate — when a product isn't huge yet, this can read as "founder grinding for traction," the wrong signal for the credibility story.

## 4. Content types (priority order)

1. **Build-in-public / case studies** — highest leverage. How we built Shelf, Vizidex, the WooCommerce agent, the listening routine. Real mechanics, real dead ends. *Ready example: the overnight customer-discovery agent — token economics, crawlability walls, the billing model nobody explains, cloud-vs-local.*
2. **Playbook / pattern content** (killdate.dev register) — the reusable methods: CLAUDE.md as an operating system, runbooks, the listening-routine pattern, "how to make AI coding tools not drift." Teaching the method = authority.
3. **Reactive POV** — a builder's anti-hype take on agentic-AI news. Cheap, steady, keeps us in the stream. (Fed by Google News RSS.)

## 5. Two motions (keep them distinct)

- **Broadcasting** (build-in-public posts) → **LinkedIn / killdate.dev.** One-to-many; builds authority.
- **Engaging** (drafted expert replies in real threads) → **the actual HN / Reddit threads** where someone asked. One-to-one; builds reputation in the room.
- They reinforce each other (people who see you helpful in a thread go check your LinkedIn, and vice versa) — but they're different surfaces.

## 6. The discovery/listening routine (agent-assisted)

*The **studio/product-side** lead-gen play — secondary to the embedded-engineer push, which is won by build documentation + network (see §3, §8), not by scanning threads.*

- **Same pattern as the WC Diagnostic agent:** scheduled agent → crawl public signal → **draft** → human acts.
- **Crawlable sources:** Google News RSS (journalist/article discovery, any topic), Hacker News via Algolia API (dev/founder/competitor-launch signal — prime channel for our audience).
- **Not crawlable (manual-watch links only):** Reddit, X/Twitter (hard-blocked), Stack Overflow. Brave MCP is dead (invalid token).
- **Hard rule for a studio/consultancy:** lead with **value/authority, never pitches.** "Hey we can build that" gets nuked as spam instantly. Drop genuine expertise → people DM you. **Authority → inbound**, not outbound conversation-injection.

## 7. Operating rules (hard)

- **Draft → human posts.** No auto-posting anywhere (ToS-risky; no connectors wired for it). The agent drafts; you publish.
- **Value-first.** No drive-by link drops, no identical mass replies. Disclose affiliation.
- Read-only on the discovery side; you approve everything that goes out.

## 8. The strategic frame (why this shape)

- **A staged portfolio:** embedded-eng + Langmerge services = **the floor + cash**; killdate.dev = **credibility engine**; the products (Shelf, Vizidex, …) = **uncapped upside / shots on goal**; content = **distribution layer**.
- **PMF first, then concentrate.** You can't will a venture-scale startup into being. Keep products *cheap to keep breathing*, fund the runway with services, watch for the **one** that pulls — then have the guts to drop the rest and pour fuel in.
- **Portfolio now, conviction later.** The only real risk is equal-weighting everything forever and never concentrating when something works.
- **For the embedded-engineer path specifically:** credibility = **technical proof** (killdate.dev, GitHub, build write-ups), won by **network + eng-leader trust**, not broadcast marketing. The listening-agent lead-gen does NOT apply to embedded gigs — that's relationship-driven.
